3 # Foaming simulation
4 Collection of software tools for the simulation of polyurethane (PU) foaming
5 process. Based on a recipe the model predicts the evolution of chemical
6 kinetics, temperature, foam density, bubble size distribution, etc.
7 This directory includes the foaming process in zero dimensional and three
8 dimensional spaces. Additionally, a wall drainage simulation can be performed, which estimates the size and shape of strut and wall thickness profile. In order to run a test case, firstly the source codes should
9 be compiled. The script `build` compiles the necessary models. The surrogate
10 models will be loaded into the database by executing the two scripts provided:
11 `initModels` and `initBubbleGrowth`. Finally, the scripts `workflow*` run the
12 simulations and detailed model for bubble growth.

14 # Dependencies and Installation
15 Several models need to be compiled. C++ and Fortran compilers are required.
16 First the MoDeNa framework should be compiled.
17 The [following steps](https://github.com/MoDeNa-EUProject/MoDeNa) describe how
18 to install MoDeNa framework.
19 
20 ### 1. Install OpenFOAM
21 The CFD solver is compiled with OpenFOAM 4.0 on Ubuntu 16.04. In order to install OpenFOAM, you can either follow the steps in [OpenFOAM](http://openfoam.org/download/4-0-ubuntu/) website or
22 copy/paste the commands below which install OpenFOAM and ParaView:
23 
24 ```
25 sudo add-apt-repository http://dl.openfoam.org/ubuntu
26 sudo sh -c "wget -O - http://dl.openfoam.org/gpg.key | apt-key add -"
27 sudo apt-get update
28 sudo apt-get -y install openfoam4
29 mkdir -p $FOAM_RUN
30 mkdir -p $FOAM_USER_APPBIN
31 mkdir -p $FOAM_USER_LIBBIN
32 ```
33 *Note 1*: This [video](https://www.youtube.com/watch?v=x6BWTEcuh38) demonstrates the installation process.
34 
35 *Note 2*: Consult with [official webpage](http://openfoam.org/download/4-0-ubuntu/) for installation problems.

128 # Run
129 In order to execute the program, you need to first provide the input variables
130 for all the modelling tools. In the `examples` directory several cases have
131 been provided. One can copy the `inputs` directory into the `foamExpansion` and
132 modify it for the targeted recipe. The `unifiedInput.json` provides the input
133 data for the bubble growth model and zero dimensional simulation, `wallDrainage_inputs.json` provides the inputs for the wall drainage simulation, `init_foamConductivity.json` provides initial points for the initialization of foamConductivity and the rest of input directory
134 creates the test case for OpenFOAM simulation. The details of the input
135 variables have been elaborated in `INPUTS.md`. After preparing the inputs the following
136 steps should be executed:
137 
138 1. Initialize surrogate nano-scale models:
139 ```
140 ./initModels
141 ```
142 Note that foamConductivity takes initial points from `init_foamConductivity.json`. This file can be prepared using `prep_init_foamConductivity.py`.
143 2. Execute meso-scopic bubble growth simulation:
144 ```
145 ./workflow_bubbleGrowth
146 ```
147 3. Initialize surrogate bubble growth model:
148 ```
149 ./initBubbleGrowth
150 ```
151 4. Run macroscopic simulation (0D or 3D):
152 ```
153 ./workflow_0D
154 ```
155 or
156 ```
157 ./workflow_3D
158 ```
159  *Note:*
160 
161  - In case of any changes in the input files steps 2-4 should repeated.
162 
163  - Initial moments of the bubble size distribution can be calculated by running
164  `initMoments`. This script uses the mean, variance and initial number
165  density of bubbles from `unifiedInput.json`.
166 
167  - If the following error occurs:
168  ```
169  --> FOAM FATAL ERROR:
170  Wrong number of arguments, expected 0 found 1
171 
172  FOAM exiting
173 
174  cannot find system Renviron
175  Fatal error: unable to open the base package
176  ```
177  The workaround is to set the R environment variable as below:
178  ```
179  R_HOME=/usr/lib/R (where you have installed R)
180  export R_HOME=/usr/lib/R
181  ```
182 
183 - The wall drainage simulation can be executed anytime after running the `./workflow_bubbleGrowth` by:
184 ```
185 ./workflow_wallDrainage
186 ```

188 # Results
189 Results of the last simulation are stored in the corresponding sub-directory of
190 the `results` directory. We use VEUSZ to visualize some of the
191 results. For example, to look at selected results for a 0D
192 simulation, you can open `plotQmom0D.vsz` using VEUSZ. Further, the results of
193 the 3D simulation (stored in the launcher directory) can be displayed using a
194 third party software such as [paraview.](http://www.paraview.org/) Additional results can be displayed through python scripts like `plotWallDrainage.py`.
195 
196 The Foam aging and Foam conductivity applications can read the final foam morphology from the results of Foam expansion. It is communicated through the `after_foaming.txt` file. In case of 3D simulation, average foam morphology is used, but user must specify several points in `system/probes` file.
