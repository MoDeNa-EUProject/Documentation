3 Zero-D example of the CFD code

6 ## Description
7 This is a zero dimensional prototype of the application of MoDeNa interface library to connect different
8 modeling scales. The first prototype shows the successful connections of nano, meso and macro sclaes for the simulation of foaming process.
9 The macro-scale code builds the RHS of 20 ODEs within the `QmomKinetics( const state_type &y , state_type &dydt , double t )` function including:
10 - dydt[0] : XW, conversion of the blowing reaction
11 - dydt[1] : XOH, conversion of the gelling reaction
12 - dydt[2] : T, temperature of the foam, K
13 - dydt[3] : L_l, weight fraction of the blowing agent in the liquid
14 - dydt[4] : L_g, weight fraction of the blowing agent in the gas
15 - dydt[5] : CO2_l, weight fraction of CO2 in the liquid
16 - dydt[6] : CO2_g, weight fraction of CO2 in the gas
17 - dydt[7] : m0, moment of order zero of the BSD
18 - dydt[8] : m1, moment of order one of the BSD
19 - dydt[9] : m2, moment of order two of the BSD
20 - dydt[10]: m3, moment of order three of the BSD
21 - dydt[11]: EG_NCO, Concentration of NCO end groups
22 - dydt[12]: EG_OH, Concentration of OH end groups
23 - dydt[13]: H2O, Water concentration
24 - dydt[14]: CO2, CO2 Concentration
25 - dydt[15]: PENTANE, Cylcopentane concentration
26 - dydt[16]: POLYMER, Dummy concentration of polymer
27 - dydt[17]: POLYMERBLOW, Second dummy concentration of polymer
28 - dydt[18]: UREA, Concentration of urea end groups
29 - dydt[19]: R_1_temp, Temperature
30 
31 Throughout the computation of RHSs the inputs of the surrogate models are set and the outputs are retirieved using modena library.
32 It should be remembred that the surrogate models should be initialized on the local machine before running this code.
33 The `write_kinetics(y, t)` function appends the results into the different text files that can be later plotted.
34 This function also computes the density of the foam based on the moments of BSD and converts the moments based
35 on the unit volume of the foam.
36 The `main(int argc, char **argv)` function starts with the initialization of the solution. Then, a 'stepper' has been
37 used to solve the ODEs. Further details on the integration method can be found [here.](http://headmyshoulder.github.io/odeint-v2/doc/boost_numeric_odeint/odeint_in_detail/steppers.html)

40 ## How to run independently?
41 
42 1. Get Boost C++ Library
43  * Skip this step if you have boost at /usr/local/ (I have used boost_1_57_0)
44  * If not, get boost:
45  \verbatim sudo apt-get install libboost-dev
46  \endverbatim
47 2. Compile the project:
48  * \verbatim cmake . \endverbatim
49 3. Make the executatble from the main code
50  * \verbatim make \endverbatim
51 4. Set the environmental variable for the shared libraries by:
52  * \verbatim export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:./eigen \endverbatim
53 5. Run the executable by:
54  * \verbatim ./QmomKinetics \endverbatim
55 6. Plot the results by software your choice.
56 
57 ### Note:
58  In modifications of the solver are required,
59  first run the cleanupScript.sh (./cleanupScript) to remove the old results
60  then compile the modified code (make),
61  run the executable (./QmomKinetics) and plot the results (gnuplot gnuplot_script.gnu).
